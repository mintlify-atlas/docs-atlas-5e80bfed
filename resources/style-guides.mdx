A collection of Swift and Objective-C style guides from prominent companies and organizations in the iOS development community.

<Tabs>
  <Tab title="Swift">
    ## Swift style guides

    ### Ray Wenderlich Swift Style Guide

    The [raywenderlich.com Swift Style Guide](https://github.com/raywenderlich/swift-style-guide) is the official Swift style guide for raywenderlich.com, one of the most widely referenced Swift style documents. It covers naming conventions, code organization, comments, and formatting. Well-maintained and frequently updated to reflect new Swift versions.

    **Highlights:**
    - Extensive naming conventions
    - Closures and function types
    - Memory management and `self` usage
    - Code organization with `// MARK:`
    - Comments and documentation

    ---

    ### GitHub Swift Style Guide

    [GitHub's Swift Style Guide](https://github.com/github/swift-style-guide) reflects how Swift is written in GitHub's own iOS applications. It is concise and pragmatic, focusing on practical conventions rather than exhaustive rules.

    **Highlights:**
    - Whitespace and formatting
    - Optionals
    - Struct vs class usage
    - Access control

    ---

    ### SlideShare Swift Style Guide

    [SlideShare's Swift Style Guide](https://github.com/SlideShareInc/swift-style-guide/blob/master/swift_style_guide.md) was developed for their iOS 8 app written in Swift. It provides a practical set of conventions used in production code.

    ---

    ### Prolific Interactive Style Guide

    [Prolific Interactive's Swift Style Guide](https://github.com/prolificinteractive/swift-style-guide) reflects conventions from an iOS agency with extensive production experience. Balanced between strictness and pragmatism.

    ---

    ### LinkedIn Swift Style Guide

    [LinkedIn's Official Swift Style Guide](https://github.com/linkedin/swift-style-guide) represents conventions used across LinkedIn's iOS codebases. Detailed and comprehensive, aimed at large-scale team development.

    ---

    ### Futurice iOS Good Practices

    [Futurice iOS Good Practices](https://github.com/futurice/ios-good-practices) is more than a style guide — it's a starting guide covering project setup, architecture, testing, and deployment in addition to code style. Maintained by Futurice developers.

    ## Objective-C style guides

    ### NY Times Objective-C Style Guide

    The [NY Times Objective-C Style Guide](https://github.com/NYTimes/objective-c-style-guide) reflects coding conventions used at The New York Times. One of the most widely cited Objective-C style documents. Covers spacing, conditionals, error handling, and more.

    **Highlights:**
    - Dot notation
    - Spacing and indentation
    - Conditionals and ternary operators
    - Error handling
    - CGRect functions

    ---

    ### raywenderlich Objective-C Style Guide

    [raywenderlich's Objective-C Style Guide](https://github.com/raywenderlich/objective-c-style-guide) outlines coding conventions for raywenderlich.com tutorials. Consistent with the site's Swift style guide in approach and thoroughness.

    **Highlights:**
    - Language conventions
    - Code organization
    - Comments
    - Naming
    - Properties

    ---

    ### GitHub Objective-C Style Guide

    [GitHub's Objective-C Style Guide](https://github.com/github/objective-c-style-guide) documents the style and conventions used in GitHub's Objective-C code. Concise and focused on the most impactful rules.

    ---

    ### Spotify Objective-C Coding Style

    [Spotify's iOS Coding Style](https://github.com/spotify/ios-style) documents guidelines for iOS development at Spotify. Reflects conventions that scale well in large engineering organizations.

## Enforcement tools

Style guides are most useful when enforced automatically. The following tools help maintain consistent style:

- [SwiftLint](https://github.com/realm/SwiftLint) — A tool to enforce Swift style and conventions. Integrates with Xcode and CI pipelines. Supports custom rules.
- [SwiftFormat](https://github.com/nicklockwood/SwiftFormat) — A code library and command-line formatting tool for reformatting Swift code.
- [OCLint](https://github.com/oclint/oclint) — Static code analysis tool for improving quality and reducing defects in Objective-C.
- [spacecommander](https://github.com/square/spacecommander) — Commit fully-formatted Objective-C code as a team without effort.
- [Tailor](https://github.com/sleekbyte/tailor) — Cross-platform static analyzer for Swift that helps you to write cleaner code and avoid bugs.
