iOS has a broad ecosystem of testing tools. This page covers behavior-driven development frameworks, UI automation tools, A/B testing, snapshot testing, network mocking, and code injection for faster iteration.

Quick and Nimble are almost always used together and represent the standard BDD stack for Swift projects.

```swift Quick + Nimble example
import Quick
import Nimble

class DolphinSpec: QuickSpec {
    override func spec() {
        describe("a dolphin") {
            var dolphin: Dolphin!
            beforeEach {
                dolphin = Dolphin()
            }
            context("when near water") {
                it("can swim") {
                    expect(dolphin.canSwim).to(beTrue())
                }
                it("has a name") {
                    expect(dolphin.name).toNot(beEmpty())
                }
            }
        }
    }
}
```

Additional BDD and unit testing tools:

- [SwiftCheck](https://github.com/typelift/SwiftCheck) — QuickCheck for Swift. Property-based testing with automatic counterexample generation.
- [Sleipnir](https://github.com/railsware/Sleipnir) — BDD-style framework for Swift.
- [Spry](https://github.com/Quick/Spry) — A Mac and iOS Playgrounds unit testing library based on Nimble.
- [swift-corelibs-xctest](https://github.com/apple/swift-corelibs-xctest) — The XCTest Project, a Swift core library for unit test support.
- [PlaygroundTDD](https://github.com/WhiskerzAB/PlaygroundTDD) — Small library to easily run your tests directly within a Playground.
- [XcodeCoverage](https://github.com/jonreid/XcodeCoverage) — Code coverage for Xcode projects.

## UI testing tools

Tools for functional testing, UI automation, and end-to-end test execution.

- [EarlGrey](https://github.com/google/EarlGrey) — iOS UI Automation Test Framework from Google. Enables synchronization with the app's UI thread.
- [KIF](https://github.com/kif-framework/KIF) — An iOS functional testing framework using accessibility labels to drive interactions.
- [appium](http://appium.io/) — Open source test automation framework for native, hybrid, and mobile web apps.
- [Bluepill](https://github.com/linkedin/bluepill) — Reliable iOS testing tool that runs UI tests using multiple simulators on a single machine.
- [AutoMate](https://github.com/PGSSoft/AutoMate) — XCTest extensions for writing UI automation tests more expressively.
- [ViewInspector](https://github.com/nalexn/ViewInspector) — Runtime inspection and unit testing of SwiftUI views.
- [TouchVisualizer](https://github.com/morizotter/TouchVisualizer) — Lightweight touch visualization library in Swift. Visualize your touches with a single line of code.
- [Remote](https://github.com/johnno1962/Remote) — Control your iPhone from inside Xcode for end-to-end testing.
- [UITestHelper](https://github.com/evermeer/UITestHelper) — UITest helper library for creating readable and maintainable tests.
- [LayoutTest-iOS](https://github.com/linkedin/LayoutTest-iOS) — Write unit tests that test the layout of a view in multiple configurations.
- [Marathon Runner](https://github.com/MarathonLabs/marathon) — Fast, platform-independent test runner focused on performance and stability.
- [Subliminal](https://github.com/inkling/Subliminal) — An understated approach to iOS integration testing.
- [Cribble](https://github.com/maxsokolov/Cribble) — Swifty tool for visual testing iPhone and iPad apps.

## A/B testing

Frameworks for running experiments and feature flags in your app.

- [Switchboard](https://github.com/KeepSafe/Switchboard) — Easy and lightweight A/B testing for mobile. Minimal server infrastructure required.
- [SkyLab](https://github.com/mattt/SkyLab) — Multivariate and A/B testing for iOS and Mac.
- [MSActiveConfig](https://github.com/mindsnacks/MSActiveConfig) — Remote configuration and A/B testing framework for iOS.
- [ABKit](https://github.com/recruit-mp/ABKit) — A/B testing framework for iOS.

## Snapshot testing

Snapshot testing captures a reference rendering of your UI and fails the test when the output changes.

- [SnapshotTesting](https://github.com/pointfreeco/swift-snapshot-testing) — Delightful Swift snapshot testing from Point-Free. Supports views, view controllers, strings, JSON, and more.
- [iOS Snapshot Test Case](https://github.com/uber/ios-snapshot-test-case) — Snapshot test your UIViews and CALayers on iOS and tvOS. Originally from Facebook.
- [SnappyTestCase](https://github.com/tooploox/SnappyTestCase) — iOS Simulator type-agnostic snapshot testing built on top of FBSnapshotTestCase.
- [second_curtain](https://github.com/ashfurrow/second_curtain) — Upload failing iOS snapshot test cases to S3 for review.
- [Playbook](https://github.com/playbook-ui/playbook-ios) — A library for isolated development of UI components with automatic snapshots.
- [MirrorDiffKit](https://github.com/Kuniwak/MirrorDiffKit) — Pretty diff between any structs or classes.

## Network mocking

Replace live network calls with controlled stubs during testing to make tests fast and deterministic.

- [OHHTTPStubs](https://github.com/AliSoftware/OHHTTPStubs) — Stub your network requests easily. Test with fake network data and custom response times, status codes, and headers.
- [Mockingjay](https://github.com/kylef/Mockingjay) — An elegant library for stubbing HTTP requests with ease in Swift.
- [DVR](https://github.com/venmo/DVR) — Network testing for Swift. Record and replay network interactions.
- [Vinyl](https://github.com/Velhotes/Vinyl) — Network testing à la VCR in Swift.
- [Kakapo](https://github.com/devlucky/Kakapo) — Dynamically mock server behaviors and responses in Swift.
- [Mockingbird](https://github.com/Farfetch/mockingbird) — Simplify software testing by easily mocking any system using HTTP/HTTPS.

## Mocking frameworks

- [Cuckoo](https://github.com/Brightify/Cuckoo) — First boilerplate-free mocking framework for Swift.

- [Fakery](https://github.com/vadymmarkov/Fakery) — Swift fake data generator for realistic test fixtures.
- [DataFixture](https://github.com/andreadelfante/DataFixture) — Create data model fixtures easily, with no headache.
