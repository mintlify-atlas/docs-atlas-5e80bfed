This page covers libraries for building augmented reality experiences with ARKit, virtual reality playback, and machine learning / CoreML model integration on iOS.

<AccordionGroup>

  <Accordion title="ARKit frameworks">

    Libraries that extend ARKit for GPS-anchored content, video recording, head-mounted display support, and persistent AR sessions.

    ```swift ARKit-CoreLocation example
    // Combine GPS coordinates with AR positioning
    let locationNode = LocationAnnotationNode(location: nil, image: pinImage)
    locationNode.scaleRelativeToDistance = true
    locationManager.addLocationNodeWithConfirmedLocation(locationNode: locationNode)
    ```

    <Card
      title="ARKit-CoreLocation"
      icon="location-dot"
      href="https://github.com/ProjectDent/ARKit-CoreLocation"
    >
      Combines the high accuracy of AR with the scale of GPS data. Lets you place AR annotations at real-world GPS coordinates, scaling content relative to distance from the user.
    </Card>

    <Card
      title="ARHeadsetKit"
      icon="glasses"
      href="https://github.com/philipturner/ARHeadsetKit"
    >
      High-level framework for using a $5 Google Cardboard headset to replicate a Microsoft HoloLens-like AR experience. Includes interactive tutorials for getting started with AR.
    </Card>

    <Card
      title="ARVideoKit"
      icon="video"
      href="https://github.com/AFathi/ARVideoKit"
    >
      Record and capture ARKit sessions as videos, photos, Live Photos, and GIFs. Supports both front and rear cameras.
    </Card>

    <Card
      title="Placenote"
      icon="map-pin"
      href="https://github.com/Placenote/PlacenoteSDK-iOS"
    >
      Makes ARKit sessions persistent to a physical location using advanced computer vision. Users can return to a location and see the same AR content anchored in place.
    </Card>

    <Card
      title="SmileToUnlock"
      icon="face-smile"
      href="https://github.com/rsrbk/SmileToUnlock"
    >
      Uses ARKit Face Tracking to detect a user's smile and trigger unlock or other in-app events. Demonstrates how to work with ARKit's facial expression blendshapes.
    </Card>

    <Card
      title="ARKit Virtual Objects"
      icon="cube"
      href="https://github.com/ignacio-chiazzo/ARKit"
    >
      Reference implementation for placing virtual objects in augmented reality. Covers plane detection, hit testing, and object placement best practices.
    </Card>

    <Card
      title="ARKit Emperor"
      icon="crown"
      href="https://github.com/kboy-silvergym/ARKit-Emperor"
    >
      A collection of the most practical ARKit sample code. Covers image tracking, face tracking, world tracking, and object detection examples.
    </Card>

    ### Key ARKit features these libraries wrap

    | Feature | Library |
    |---------|---------|
    | World tracking + GPS overlay | [ARKit-CoreLocation](https://github.com/ProjectDent/ARKit-CoreLocation) |
    | Face tracking (blend shapes) | [SmileToUnlock](https://github.com/rsrbk/SmileToUnlock) |
    | Session recording | [ARVideoKit](https://github.com/AFathi/ARVideoKit) |
    | Persistent anchors | [Placenote](https://github.com/Placenote/PlacenoteSDK-iOS) |
    | Head-mounted display rendering | [ARHeadsetKit](https://github.com/philipturner/ARHeadsetKit) |
    | Object placement / hit testing | [ARKit Virtual Objects](https://github.com/ignacio-chiazzo/ARKit) |

  </Accordion>

  <Accordion title="VR libraries">

    Libraries for 360-degree video playback and basic virtual reality experiences on iOS.

    <Note>
      Apple's platform does not have a dedicated VR SDK. These libraries provide 360° video players and basic cardboard-style VR viewing, often built on top of SceneKit or Metal.
    </Note>

    <Card
      title="360 VR Player"
      icon="circle-play"
      href="https://github.com/hanton/HTY360Player"
    >
      Open source, ad-free, native and universal 360-degree panorama video player for iOS. Supports both equirectangular and spherical projections.
    </Card>

    <Card
      title="Swifty360Player"
      icon="arrows-rotate"
      href="https://github.com/abdullahselek/Swifty360Player"
    >
      iOS 360-degree video player streaming from an AVPlayer, written in Swift. Supports head movement via device gyroscope.
    </Card>

    <Card
      title="VR Toolkit iOS"
      icon="vr-cardboard"
      href="https://github.com/Aralekk/VR_Toolkit_iOS"
    >
      Sample project that provides the basics for creating an interactive VR experience on iOS. Good starting point for cardboard-style stereo rendering.
    </Card>

    <Card
      title="simple360player"
      icon="film"
      href="https://github.com/Aralekk/simple360player_iOS"
    >
      Free and ad-free 360 VR video player. Supports flat and stereoscopic display modes. Written in Swift.
    </Card>

  </Accordion>

  <Accordion title="Machine learning libraries">

    Libraries for CoreML model integration, neural networks, deep learning, and on-device inference.

    **CoreML & Apple frameworks**

    | Library | Description |
    |---------|-------------|
    | [CoreML-Models](https://github.com/likedan/Awesome-CoreML-Models) | A curated collection of unique Core ML models ready to drop into iOS apps. |
    | [CoreML-samples](https://github.com/ytakzk/CoreML-samples) | Sample code for Core ML using ResNet50 (provided by Apple) and custom models generated by coremltools. |
    | [SwiftCoreMLTools](https://github.com/JacopoMangiavacchi/SwiftCoreMLTools) | A Swift library for creating and exporting CoreML models programmatically in Swift. |
    | [iOS-GenAI-Sampler](https://github.com/shu223/iOS-GenAI-Sampler) | A collection of Generative AI examples on iOS, covering diffusion models and LLMs. |

    **Deep learning frameworks**

    | Library | Description |
    |---------|-------------|
    | [Tensorflow-iOS](https://github.com/tensorflow/tensorflow/tree/master/tensorflow/examples/ios) | The official Google TensorFlow neural network library port for iOS. |
    | [Bender](https://github.com/xmartlabs/Bender) | Craft fast neural networks using TensorFlow models with Metal under the hood. |
    | [Serrano](https://github.com/pcpLiu/Serrano) | A deep learning library for iOS and macOS with GPU acceleration. |
    | [DL4S](https://github.com/palle-k/DL4S) | Deep Learning for Swift: accelerated tensor operations and dynamic neural networks based on reverse mode automatic differentiation. |
    | [TensorSwift](https://github.com/qoncept/TensorSwift) | A lightweight library to calculate tensors in Swift with TensorFlow-like APIs. |

    **General ML & AI**

    | Library | Description |
    |---------|-------------|
    | [Swift-AI](https://github.com/Swift-AI/Swift-AI) | The Swift machine learning library. Covers neural networks and common ML algorithms. |
    | [Swift-Brain](https://github.com/vlall/Swift-Brain) | AI/ML data structures and Swift algorithms. Includes Bayes theorem and neural network implementations. |
    | [AIToolbox](https://github.com/KevinCoble/AIToolbox) | A toolbox of AI modules in Swift: graphs/trees, linear regression, SVMs, neural networks, PCA, KMeans, genetic algorithms, and MDP. |
    | [Revolver](https://github.com/petrmanek/Revolver) | A framework for building fast genetic algorithms in Swift, with modular architecture and pre-implemented operators. |

    <Tip>
      For production on-device inference, Apple's CoreML framework (via `CoreML-Models` or `SwiftCoreMLTools`) is the recommended approach as it takes advantage of the Neural Engine on A-series chips.
    </Tip>

  </Accordion>
