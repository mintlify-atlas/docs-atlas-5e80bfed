## Database & ORM

Full-featured database engines and object-relational mappers for iOS.

<Tip>Realm is the most developer-friendly option for mobile-first persistence. SQLite.swift and GRDB.swift are excellent choices when you need SQL-level control.</Tip>

### Getting started with Realm

```swift Realm object and write example
import RealmSwift

class Dog: Object {
    @Persisted var name: String
    @Persisted var age: Int
}

// Write
let realm = try! Realm()
let dog = Dog()
dog.name = "Rex"
dog.age = 3
try! realm.write {
    realm.add(dog)
}

// Query
let dogs = realm.objects(Dog.self).filter("age > 1")
for d in dogs {
    print("\(d.name) is \(d.age) years old")
}
```

### Getting started with SQLite.swift

```swift SQLite.swift table and query
import SQLite

let db = try Connection("path/to/db.sqlite3")

let users = Table("users")
let id    = Expression<Int64>("id")
let name  = Expression<String>("name")
let email = Expression<String?>("email")

try db.run(users.create { t in
    t.column(id, primaryKey: true)
    t.column(name)
    t.column(email, unique: true)
})

// Insert
let insert = users.insert(name <- "Alice", email <- "alice@example.com")
let rowid  = try db.run(insert)

// Query
for user in try db.prepare(users) {
    print("id: \(user[id]), name: \(user[name])")
}
```

## UserDefaults extensions

Type-safe, ergonomic wrappers around `NSUserDefaults` / `UserDefaults`.

<Note>Apple's `UserDefaults` API is untyped. These libraries add compile-time safety and modern Swift syntax.</Note>

```swift SwiftyUserDefaults example
import SwiftyUserDefaults

// Define keys with types
extension DefaultsKeys {
    var username: DefaultsKey<String?> { .init("username") }
    var launchCount: DefaultsKey<Int>  { .init("launchCount", defaultValue: 0) }
    var isPremium: DefaultsKey<Bool>   { .init("isPremium", defaultValue: false) }
}

// Read
let name  = Defaults[\.username]
let count = Defaults[\.launchCount]

// Write
Defaults[\.username]    = "alice"
Defaults[\.launchCount] += 1
```
