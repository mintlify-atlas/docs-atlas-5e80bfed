Writing Auto Layout constraints in raw Swift or Objective-C is verbose. These libraries give you concise, readable syntax while staying close to the native APIs.

## Auto Layout DSLs

```swift SnapKit
import SnapKit

view.addSubview(button)
button.snp.makeConstraints { make in
    make.centerX.equalToSuperview()
    make.bottom.equalTo(view.safeAreaLayoutGuide).inset(20)
    make.width.equalTo(200)
    make.height.equalTo(44)
}
```

```swift PureLayout
import PureLayout

imageView.autoPinEdgesToSuperviewEdges(with: UIEdgeInsets(top: 0, left: 16, bottom: 0, right: 16))
imageView.autoSetDimension(.height, toSize: 200)
```

```swift Cartography
import Cartography

constrain(button, view) { button, view in
    button.centerX == view.centerX
    button.bottom == view.bottom - 20
    button.width == 200
    button.height == 44
}
```

## Flexbox

<CardGroup cols={2}>
  <Card title="FlexLayout" icon="flex" href="https://github.com/layoutBox/FlexLayout">
    Gently wraps Facebook's Yoga flexbox engine with concise, intuitive, chainable syntax
  </Card>
  <Card title="YogaKit" icon="yoga" href="https://github.com/facebook/yoga/tree/master/YogaKit">
    Facebook's powerful Yoga layout engine (Flexbox) for iOS
  </Card>
</CardGroup>

```swift FlexLayout
import FlexLayout

rootFlexContainer.flex.direction(.column).padding(12).define { flex in
    flex.addItem(imageView).width(100%).aspectRatio(1.5)
    flex.addItem().direction(.row).define { flex in
        flex.addItem(nameLabel).grow(1)
        flex.addItem(priceLabel)
    }
}
```

## Manual & frame-based layout

For performance-critical views (like cells with many subviews), manual layout is sometimes the right choice:
